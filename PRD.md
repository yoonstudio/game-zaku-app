# PRD: Zaku Colony Destroyer (자쿠 콜로니 파괴 게임)

**문서 버전**: 1.0
**작성일**: 2026-01-23
**상태**: Draft

---

## 1. Executive Summary

### 1.1 제품 비전 및 미션

**비전**: 일본 애니메이션 "기동전사 건담"의 상징적인 자쿠(Zaku) 로봇을 조종하여 우주 콜로니를 파괴하는 몰입감 있는 3D 브라우저 게임을 제공한다.

**미션**: Three.js 기반의 웹 기술을 활용하여 별도 설치 없이 브라우저에서 즉시 플레이 가능한 액션 슈팅 게임을 개발하며, 건담 시리즈의 세계관을 경험할 수 있는 인터랙티브 콘텐츠를 제공한다.

### 1.2 문제 정의

- 건담 시리즈 팬들이 자쿠를 직접 조종해볼 수 있는 접근 가능한 게임이 부족함
- 브라우저 기반의 고품질 메카(Mecha) 게임 콘텐츠가 제한적임
- 간단한 조작으로 몰입감 있는 파괴 액션을 즐길 수 있는 캐주얼 게임 수요 존재

### 1.3 제안 솔루션

웹 브라우저에서 바로 실행 가능한 3D 액션 게임으로, 플레이어가 자쿠를 조종하여 우주 콜로니를 공격하고 파괴하는 미션을 수행한다. 다양한 무기 시스템과 파괴 메카닉을 통해 시각적으로 만족스러운 게임플레이를 제공한다.

### 1.4 핵심 성공 지표

| 지표 | 목표값 | 측정 방법 |
|------|--------|-----------|
| 초기 로딩 시간 | 5초 이내 | Performance API |
| 프레임 레이트 | 60 FPS 유지 | Stats.js 모니터링 |
| 평균 플레이 시간 | 10분 이상 | 세션 분석 |
| 콜로니 파괴 완료율 | 70% 이상 | 게임 이벤트 로깅 |

## 3. 사용자 분석

### 3.1 타겟 사용자 페르소나

#### 페르소나 1: 건담 팬 (태현, 25세)
- **배경**: 대학생, 건담 시리즈 열성 팬
- **목표**: 자쿠를 직접 조종하며 애니메이션 속 장면을 재현
- **통증점**: 건담 관련 게임은 대부분 콘솔 전용이거나 비쌈
- **니즈**: 무료로 바로 플레이 가능한 건담 테마 게임

#### 페르소나 2: 캐주얼 게이머 (수민, 30세)
- **배경**: 직장인, 점심시간에 가볍게 게임을 즐김
- **목표**: 스트레스 해소를 위한 파괴적인 게임플레이
- **통증점**: 설치나 회원가입이 필요한 게임은 번거로움
- **니즈**: 브라우저에서 바로 시작, 5-10분 플레이 가능

#### 페르소나 3: 메카 애호가 (준호, 35세)
- **배경**: IT 개발자, 로봇/메카 컨텐츠 수집가
- **목표**: 고품질 메카 게임 경험
- **통증점**: 웹 기반 메카 게임의 품질이 낮음
- **니즈**: 시각적으로 인상적인 3D 로봇 게임

### 4.1 자쿠 로봇 (Zaku II)

#### 4.1.1 모델 구조
```
Zaku II (루트)
├── Head (모노아이 헤드)
│   └── MonoEye (단안 센서, 발광)
├── Torso (상체)
│   ├── LeftShoulder (왼쪽 어깨 아머)
│   │   └── LeftArm (왼팔)
│   │       └── LeftHand (왼손 - 무기 장착)
│   └── RightShoulder (오른쪽 어깨 아머 + 스파이크)
│       └── RightArm (오른팔)
│           └── RightHand (오른손 - 쉴드 또는 무기)
├── Waist (허리)
├── Skirt (스커트 아머)
├── LeftLeg (왼다리)
│   └── LeftFoot (왼발 - 버니어)
├── RightLeg (오른다리)
│   └── RightFoot (오른발 - 버니어)

game_03 프로젝트에 Three.js 기반 게임 환경을 구축해줘.

요구사항:

2. src/main.js 진입점 생성
3. 기본 Three.js 씬 설정
   - 우주 배경 (검은색 + 별)
   - 기본 조명 (AmbientLight + DirectionalLight)
4. 카메라 설정 (PerspectiveCamera)
5. 반응형 캔버스 처리
6. 애니메이션 루프 구현

결과: 우주 배경이 있는 빈 3D 공간이 렌더링되어야 함.
```

### Stage 2: 자쿠 로봇 모델 생성

```
Three.js 기본 도형으로 자쿠 II 로봇을 만들어줘.

자쿠 구조:
- 머리: 구체 (모노아이 포함)
- 몸통: 박스 + 어깨 아머
- 팔: 원기둥 (좌우)
- 다리: 원기둥 (좌우)
- 백팩: 박스 (버니어 포함)

요구사항:
1. OD 그린 (0x5A5A3A) 색상 적용
2. 모노아이는 붉은색 발광 (0xFF0000)
3. 전체 높이 약 17.5 단위
4. 로봇을 별도 클래스로 분리 (Zaku.js)
5. 부모-자식 계층 구조로 관절 설정

결과: 자쿠 형태의 로봇이 우주 공간에 표시되어야 함.
```

### Stage 3: 이동 및 카메라 시스템

```
자쿠의 이동 및 카메라 시스템을 구현해줘.

이동 컨트롤:
- WASD: 전후좌우 이동
- Q/E: 좌우 회전
- Space: 상승
- Shift: 하강
- Ctrl + 이동키: 부스트 (빠른 이동)

카메라:
- 3인칭 팔로우 카메라
- 마우스로 조준 (화면 중앙에 크로스헤어)

요구사항:
1. 부드러운 이동 (관성 적용)
2. 이동 시 자쿠가 진행 방향을 바라봄
3. 카메라가 자쿠를 부드럽게 추적
4. InputManager 클래스로 입력 관리

결과: 자쿠가 우주 공간을 자유롭게 이동해야 함.
```

### Stage 4: 콜로니 및 배경 구현

```
우주 콜로니와 배경을 구현해줘.

콜로니 구조:
- 원통형 (O'Neill Cylinder)

- 창문 구역 (발광)
- 도킹 베이 구조물

- 스타필드 (수천 개의 별)
- 지구 (큰 구체, 배경에 배치)

요구사항:
1. 콜로니를 별도 클래스로 분리 (Colony.js)
2. 콜로니 각 구역에 독립적인 메시 부여
3. 구역별 내구도 속성 추가
4. 콜로니 내부 발광 효과

결과: 거대한 우주 콜로니가 우주 배경에 표시되어야 함.

### Stage 5: 무기 시스템 구현

```
자쿠의 무기 시스템을 구현해줘.

무기 목록:
1. 자쿠 머신건: 좌클릭, 연사, 데미지 10
2. 바주카: 숫자 2 또는 휠 전환, 단발, 데미지 100, 폭발 범위

요구사항:
1. WeaponSystem 클래스 구현
2. Projectile 클래스 (발사체)
3. 좌클릭으로 현재 무기 발사
4. 발사체가 조준 방향으로 이동
5. 발사 이펙트 (머즐 플래시)
6. 폭발 이펙트 (파티클)
7. 탄약 시스템 (잔탄 표시)
8. 재장전 (R키)

결과: 무기 발사 및 이펙트가 정상 동작해야 함.

### Stage 6: 충돌 및 파괴 시스템

```
발사체와 콜로니의 충돌 및 파괴 시스템을 구현해줘.

요구사항:
1. 레이캐스팅 기반 충돌 감지
2. 콜로니 구역별 내구도 감소
3. 내구도 0 시 해당 구역 파괴
4. 파괴 시:
   - 폭발 파티클
   - 잔해 생성 (우주로 흩어짐)
   - 점수 획득
5. 연쇄 폭발 (인접 구역 30% 확률)
6. 파괴된 구역 시각적 변화

결과: 콜로니를 공격하면 파괴되어야 함.
```
